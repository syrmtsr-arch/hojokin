プロジェクトの準備 Claude Codeで開く 補助金情報を見る note記事の下書き よく使う依頼例

作業マニュアル（スタッフ向け）

GitHub・Claude Codeの使い方とnote記事下書きの作り方

このマニュアルは、TENJO KANAZAWAの補助金情報プロジェクトを手伝ってもらう方向けの手順書です。 GitHubアカウントは不要、Claude Codeのインストール・ログインは済んでいる前提で説明します。

1. プロジェクトを自分のPCに用意する（最初の1回だけ）

1-1. Gitをインストール

まだの場合は Git公式サイト からインストールしてください。

1-2. プロジェクトをダウンロード（クローン）

保存したい場所（例：デスクトップ）でターミナル（PowerShellなど）を開き、以下を実行します。

git clone https://github.com/syrmtsr-arch/hojokin.git

hojokin というフォルダができ、中にプロジェクト一式がダウンロードされます。 このリポジトリは公開（パブリック）なので、ログインなしでダウンロードできます。

「No such file or directory」と出た場合（cd Desktopなどで失敗するとき）

cd Desktop のようなコマンドで「そんなファイルやフォルダはありません」と出ることがあります。 Windowsの環境によっては、デスクトップのフォルダ名が Desktop（英語）ではなく デスクトップ（日本語）になっていたり、OneDriveと同期していてフォルダの場所が変わっていることがあるためです。

その場合は、まずユーザーフォルダに移動してから進めると確実です。以下を1行ずつ入力してEnterを押してください。

cd /c/Users/あなたのユーザー名

（左側の表示が MINGW64 /c/Users/あなたのユーザー名 のように変わればOKです）

その後、デスクトップに移動したい場合は以下のいずれかを試してください。

cd Desktop
cd デスクトップ
cd OneDrive/デスクトップ

どれもうまくいかない場合は、ユーザーフォルダ直下（cd /c/Users/あなたのユーザー名の状態）のまま git clone を実行しても問題ありません。

1-3. （以後）最新の状態に更新する

作業を始める前に、毎回以下を実行して最新版を取得することをおすすめします。

cd hojokin
git pull
⚠️ 今回は「変更をGitHubに送り返す（push）」は行いません。自分のPC内で作業・保存するだけでOKです。

2. Claude Codeでプロジェクトを開く

  1. ターミナルで hojokin フォルダに移動します。 
    cd hojokin
  2. claude と入力して起動します。 
    claude
  3. 起動すると、このフォルダが「プロジェクト」として認識されます。CLAUDE.md というファイルに作業ルールが書かれており、Claudeが自動で読み込みます。

これで、私（管理者）と同じルール・フォーマットでClaudeに作業を頼める状態になります。

3. 補助金情報をHTMLで見る

note記事の下書きを作る前に、現在登録されている補助金・支援制度の情報をブラウザで確認できます。hojokin フォルダ内の以下のファイルを、エクスプローラーでダブルクリックすればブラウザで開きます。

  • raw/inbox/subsidy-guide-2026.html
    プレゼン形式（スライド風）のガイド本体。制度の概要・締切カレンダー（ガントチャート）などをまとめて見られます。

最新情報に更新したい場合は、Claudeに「補助金情報を最新化して」と依頼すると、このファイルと元データ（wiki/sources/subsidy-calendar-2026.md など）がまとめて更新されます（詳しくは「5. その他、よく使う依頼例」参照）。

3-1. Web版（公開URL）

同じ情報をWebアプリ版としても公開しています。PCを持っていない人にもこのURLを共有するだけでOKです。

https://syrmtsr-arch.github.io/hojokin/

状況・カテゴリでの絞り込みやキーワード検索ができます。中身は webapp/index.html と webapp/subsidies.json で、これらを master ブランチにpushするとGitHub Actionsが自動で再公開します（管理者のみ・通常のスタッフ作業では意識不要です）。

4. note記事の下書きを作る

TENJO KANAZAWAのnote「補助金情報」シリーズは毎月更新します。下書き作成の流れは以下の通りです。

4-1. Claudeに依頼する

Claude Codeのチャットで、たとえば以下のように頼んでください。

今月分のnote記事の下書きを作って

Claudeは以下のデータを参照して記事を作成します。

  • wiki/synthesis/note-article-improvement-2026.md … 記事の構成テンプレート・方針
  • wiki/sources/subsidy-calendar-2026.md … 締切カレンダー
  • wiki/sources/subsidy-database-ishikawa-startup-2026.md … 制度の詳細データベース

4-2. 記事の構成（テンプレート）

毎月の記事は、原則として以下の5部構成です。

  1. 今月の最重要トピック（1〜2件）
    締切が近い・金額が大きい・新規公募開始など「今月動くべき理由がある」制度を深掘り。金額・補助率・対象を必ず明記し、「こんな人向け」を一言で添える。
  2. 今月〜来月が動きどきの制度（一覧）
    カレンダーをもとに、今月公募中／来月締切のものを表で紹介（①ほど深掘りしない）。
  3. 準備しておきたい制度（先の締切）
    2〜3ヶ月先に締切が来るもの・例年この時期に公募が出るものを「今のうちに準備すると有利」という切り口で。
  4. ミニ特集（毎月テーマを変える）
    過去に紹介した制度の中から、今月のテーマに沿うものをピックアップして深掘り。
    テーマ例：能登で起業する人向け／賃上げ・人材確保系／ビジネスコンテストで挑戦する　など
  5. 相談へのCTA（固定）
    ISICO創業サポートデスク・TENJOオープンデスクなどへの案内（毎回ほぼ同じ文言）。

4-3. 各制度紹介で必ず入れる項目

  • 対象
  • 金額・補助率
  • 公募期間
  • 「こんな人向け」一言コメント
  • リンク

4-4. 下書きができたら

Claudeが作成したMarkdownファイルを開いて内容を確認・修正します。

Markdownファイル（.md）の開き方

作成された下書きは wiki/synthesis/ フォルダの中に .md ファイルとして保存されます。開き方はいくつかありますが、以下のいずれかが簡単です。

  • Claudeに表示してもらう：チャットで「さっき作った下書きの内容を見せて」と頼めば、チャット画面にそのまま表示してくれます
  • メモ帳で開く：エクスプローラーでファイルを右クリック →「プログラムから開く」→「メモ帳」（装飾は表示されませんが、文章はそのまま読めます）
  • VS Codeで開く：インストール済みであれば、ファイルをダブルクリック、またはClaude Codeのターミナルで code ファイルパス と入力すると、見出しなどがきれいに表示されたプレビューで確認できます

修正したい箇所があれば、ファイルを直接編集するか、Claudeに「この部分をこう直して」と伝えれば修正してもらえます。

  • 文章の調整は、Claudeに「もっとカジュアルに」「この制度の説明をもっと詳しく」のように指示すれば直してもらえます
  • 完成したら、note.comの編集画面にコピー＆ペーストして仕上げます

5. その他、よく使う依頼例

  • 「補助金情報を最新化して」 → 各制度の公式サイトをWeb検索で確認し、カレンダー・データベースを更新
  • 「wikiをチェックして」 → ページ間の矛盾や古い情報、孤立ページがないかチェック
